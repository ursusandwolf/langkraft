## 🔴 Остающиеся критичные моменты

### 1. Несоответствие `SyncManager` и `SqlDelightVocabularyRepository`
**Проблема:** `SyncManager` в текущем виде использует in-memory очередь `mutableListOf<PendingSyncChange>()`. В то же время `SqlDelightVocabularyRepository` и БД имеют полноценную таблицу `PendingSyncChange` для персистентности.
**Решение:** Переписать `SyncManager`, чтобы он вызывал `vocabularyRepository.sync()`. `SyncManager` должен отвечать за *когда* синхронизироваться (background, по кнопке, при восстановлении сети), а репозиторий — за *как* (DB + API).

### 2. Хардкод URL в Репозитории
**Файл:** `SqlDelightVocabularyRepository.kt`
**Проблема:** URL `https://api.langkraft.com/api/sync` зашит прямо в коде. Это мешает тестированию на локальном окружении и нарушает принципы конфигурации.
**Решение:** Прокидывать базовый URL через конструктор или использовать `Ktor` конфигурацию.

## 🟡 Архитектурные рекомендации и UX

### 3. Waveform: От симуляции к реальности
**Статус:** Сейчас бекенд генерирует случайные данные (Random). 
**Рекомендация:** Настроить вызов `audiowaveform` или `ffmpeg` на сервере. Для контейнеризированного окружения это потребует установки соответствующих бинарников.

### 4. Отсутствие обработки конфликтов при синхронизации
**Файл:** `BackendVocabularyRepository.kt`
**Проблема:** Бекенд просто делает `upsert`. Если одно и то же слово было изменено на двух устройствах, победит тот, кто синхронизировался последним (Last Write Wins). 
**Рекомендация:** Внедрить базовую логику на основе `lastUpdated` timestamp для более умного слияния.

## 🔵 Предложения по коду (Clean Code)

### 5. Дублирование логики маппинга
Маппинг из `Vocabulary` (SQLDelight) в `VocabularyWord` (Domain) встречается в нескольких местах. Стоит вынести его в Extension-функцию в `shared/src/commonMain/kotlin/com/langkraft/data/repository/Mappers.kt`.

### 6. Типизация ошибок
Рекомендуется заменить `println("Sync failed: ${e.message}")` на структурированное логирование или возвращать `Result<Long>` из метода `sync`, чтобы UI мог показать уведомление об ошибке синхронизации.
